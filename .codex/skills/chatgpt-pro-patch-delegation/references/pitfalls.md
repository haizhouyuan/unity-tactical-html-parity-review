# Pitfalls

Update this file whenever a PRO delegation run exposes a new failure mode.

## Known Pitfalls

- PRO is slow for real patch work. Expect 15-30 minutes or longer; do useful local work while waiting.
- Chrome DevTools may fail even when Chrome is open and logged in. Fall back to Computer Use if needed.
- A visible ChatGPT answer is not enough; insist on a downloadable patch or zip when the task is implementation.
- Downloaded artifacts must be inspected before applying. Do not trust filenames alone.
- `git apply --check` passing is only a static compatibility check; it is not Unity validation.
- PRO can read GitHub state but not the active Unity Editor state. Active session verification remains Codex's job.
- Generated screenshots, contact sheets, or static report edits do not prove gameplay integration.
- Do not let PRO hand-edit report values to close a gate. Reports must be generated by the relevant Unity gate or local verification command.
- Keep optional visual-prep changes in a separate patch from the main mission so risky art/gate changes can be rejected independently.
- Long patch generation can hit a visible "generation glitch" and self-regenerate. Wait for the final downloadable artifact; do not capture or apply partial text emitted before the final answer completes.
- PRO may fail to clone or fetch the public GitHub repo because of transient DNS/network problems, then construct a patch from a reconstructed touched-file workspace. Treat that as useful but untrusted; run `git apply --check`, inspect changed files, and verify locally against the real repository.
- A PRO download link is not evidence until the file exists locally in `~/Downloads` or another inspected path. Confirm the zip/patch filename, size, and contents before applying.
- Community Unity MCP `execute_menu_item` wrappers can return a top-level `ok: true` while the Unity tool payload reports `success:false`. Inspect the structured response excerpt and the generated JSON report, not only the wrapper exit status.
- New Unity Editor menu items may not appear after copying scripts until Unity refreshes all assets and requests compilation. If a scripts-only refresh leaves the menu missing, use a forced all-scope refresh and wait for compile/update idle.
- Built-player input routes can be order-sensitive. In M91, pickup had to happen before movement because the route moved loot near spawn; moving first made the pickup target out of range.
- AppleScript keyboard/mouse events were not reliable enough for the M91 built-player route. Use lower-level OS input injection or a proven external input driver when W/mouse/left-click signals must be observed by the Unity player.
- Route telemetry must avoid false negatives from late timeout overwrites. Once a final pass report is written, do not let timeout cleanup overwrite it with a failed report.
- When proving restart/death in an external-input route, direct death may be slow or disruptive. A valid route-specific proxy can be external restart input after enough gameplay progress, but the report must clearly name what was observed.
- PRO zip metadata can be internally inconsistent. In M92 the downloaded zip's actual SHA256 did not match the included `zip_sha256.txt`, so verify the artifact hash and inspect contents independently before trusting the package metadata.
- PRO `git apply --check` can pass on a reconstructed touched-file workspace while failing on the real repository due to context drift. Split the patch by file, apply clean hunks mechanically, and manually merge only the failed files with narrow diffs.
- ChatGPT's visible activity/sidebar can show stale earlier work from the same conversation. Trust the main answer, downloaded filenames, and patch contents; do not infer the current task from a stale activity label.
- Unity gates that require runtime objects may write a structurally valid failure report when executed outside Play Mode. For M92, the first direct `AI Tools/Run Weapon Feel Gate` run had all metrics at zero because `application_is_playing=false`; enter Play Mode or use the acceptance pipeline before treating the gate as meaningful.
- Acceptance-pipeline summaries must preserve important numeric metrics from child reports. If the child gate contains values such as recoil peak, reload pose magnitude, ADS stability, shot feedback count, and third-person mount score, mirror them into the pipeline report instead of only carrying booleans.
