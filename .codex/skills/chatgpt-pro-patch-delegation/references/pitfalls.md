# Pitfalls

Update this file whenever a PRO delegation run exposes a new failure mode.

## Known Pitfalls

- PRO is slow for real patch work. Expect 15-30 minutes or longer; do useful local work while waiting.
- Chrome DevTools may fail even when Chrome is open and logged in. Fall back to Computer Use if needed.
- A visible ChatGPT answer is not enough; insist on a downloadable patch or zip when the task is implementation.
- Downloaded artifacts must be inspected before applying. Do not trust filenames alone.
- `git apply --check` passing is only a static compatibility check; it is not Unity validation.
- PRO can read GitHub state but not the active Unity Editor state. Active session verification remains Codex's job.
- Generated screenshots, contact sheets, or static report edits do not prove gameplay integration.
- Do not let PRO hand-edit report values to close a gate. Reports must be generated by the relevant Unity gate or local verification command.
- Keep optional visual-prep changes in a separate patch from the main mission so risky art/gate changes can be rejected independently.
- Long patch generation can hit a visible "generation glitch" and self-regenerate. Wait for the final downloadable artifact; do not capture or apply partial text emitted before the final answer completes.
